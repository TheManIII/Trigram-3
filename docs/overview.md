# Trigram-3 架构概述

## 简介

**Trigram-3** 是一个基于平衡三进制（Balanced Ternary）的 RISC 架构 CPU 指令集。它使用 T(-1), 0, 1 作为基本数字，通过极简的 9 条指令实现完整的通用计算功能。

## 核心特性

- **平衡三进制系统**：使用 T(-1), 0, 1 作为基本数字
- **极简指令集**：仅 9 条指令覆盖通用计算
- **对称地址空间**：逻辑地址范围 -9841 到 +9841
- **内存映射 I/O**：无需新增指令即可实现输入输出
- **固定指令长度**：每条指令 27 trit（3 个内存字）

## 架构参数

| 参数 | 值 | 说明 |
|------|---|------|
| 字长（Word Size） | 9 trit | 与寄存器宽度一致 |
| 指令长度 | 27 trit | 每条指令占 3 个内存字 |
| 寄存器数量 | 9 个（R0-R8） | R0 硬连线为 0 |
| 地址空间 | -9841 到 +9841 | 对称地址空间 |
| 物理内存 | 19683 字 | 逻辑地址 + 9841 |

## 指令集概览

| Opcode | 助记符 | 操作 | 说明 |
|--------|--------|------|------|
| 000 | HALT | 停机 | 停止执行 |
| 001 | ADD | Rd ← Rs1 + Rs2 | 两寄存器相加 |
| 00T | OFFSET | Rd ← Rs1 + Imm₉ | 加立即数 |
| 010 | LOAD | Rd ← Mem[Rs1 + Imm₉] | 读内存 |
| 0T0 | STORE | Mem[Rs1 + Imm₉] ← Rs2 | 写内存 |
| 100 | CMP | Rd ← sign(Rs1 - Rs2) | 比较 |
| T00 | TJUMP | 根据Rd值三路跳转 | 条件跳转 |
| 01T | ABS | Rd ← \|Rs1\| | 绝对值 |
| 0T1 | NEG | Rd ← -Rs1 | 取负 |

## 寄存器模型

| 寄存器 | 编码（3 trit） | 说明 |
|--------|---------------|------|
| R0 | 000 | 零号寄存器，硬连线为 0 |
| R1 | 001 | 通用寄存器 |
| R2 | 01T | 通用寄存器 |
| R3 | 010 | 通用寄存器 |
| R4 | 011 | 通用寄存器 |
| R5 | 1TT | 通用寄存器 |
| R6 | 1T0 | 通用寄存器 |
| R7 | 1T1 | 通用寄存器 |
| R8 | 10T | 通用寄存器 |

## 内存模型

### 地址空间

- **逻辑地址**：-9841 到 +9841
- **物理地址**：逻辑地址 + 9841（0 到 19682）
- **特殊地址**：
  - -9841：输出端口
  - -9840：输入端口
  - 0：程序入口

### 内存映射 I/O

Trigram-3 不增加任何 I/O 专用指令，而是通过内存映射 I/O 实现：

```assembly
; 从输入端口读取
LOAD R1, [-9840]

; 向输出端口写入
STORE R1, [-9841]
```

## 指令格式

每条指令固定 27 trit（3 个内存字）：

```
Opcode(3) | Rd(3) | Rs1(3) | Imm(18)
```

对于 ADD、CMP、STORE 指令，Imm 的低 3 trit 包含 Rs2。

## 启动与执行

- **程序入口**：逻辑地址 0
- **指令对齐**：所有指令必须从地址 0, 3, 6, 9... 开始
- **执行流程**：取指 → 译码 → 执行
- **停机**：执行 HALT 指令或遇到异常

## 异常处理

Trigram-3 采用简单的异常处理机制：

- **非法指令**：CPU 停机并输出错误信息
- **无效地址**：CPU 停机并输出错误信息
- **其他错误**：CPU 停机

## 调试支持

Trigram-3 提供完整的调试支持：

- **单步执行**：逐条执行指令
- **断点设置**：在指定地址暂停
- **状态查看**：查看寄存器和内存状态
- **反汇编**：将机器码反汇编为汇编代码
