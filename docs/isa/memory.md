# Trigram-3 内存模型

## 内存概览

Trigram-3 采用对称的地址空间设计，逻辑地址范围为 -9841 到 +9841。

## 内存参数

| 参数 | 值 | 说明 |
|------|---|------|
| 字长（Word Size） | 9 trit | 与寄存器宽度一致 |
| 逻辑地址范围 | -9841 到 +9841 | 对称地址空间 |
| 物理内存大小 | 19683 字 | 逻辑地址 + 9841 |
| 物理地址范围 | 0 到 19682 | 逻辑地址转换后 |

## 地址转换

### 逻辑地址到物理地址

```
物理地址 = 逻辑地址 + 9841
```

示例：
- 逻辑地址 -9841 → 物理地址 0
- 逻辑地址 0 → 物理地址 9841
- 逻辑地址 +9841 → 物理地址 19682

### 物理地址到逻辑地址

```
逻辑地址 = 物理地址 - 9841
```

## 特殊地址

| 逻辑地址 | 用途 | 说明 |
|----------|------|------|
| -9841 | 输出端口 | 写入此地址输出到控制台 |
| -9840 | 输入端口 | 读取此地址从控制台输入 |
| 0 | 程序入口 | 程序开始执行的地址 |

## 内存映射 I/O

Trigram-3 不增加任何 I/O 专用指令，而是通过内存映射 I/O 实现。

### 输入

```assembly
LOAD R1, [-9840]  ; 从输入端口读取到 R1
```

### 输出

```assembly
STORE R1, [-9841] ; 将 R1 写入输出端口
```

## 内存访问

### 读内存 (LOAD)

```assembly
LOAD Rd, [Rs1, #imm]
```

操作：`Rd ← Mem[Rs1 + imm]`

示例：
```assembly
OFFSET R1, R0, #10   ; R1 = 10
LOAD R2, [R1, #5]     ; R2 = Mem[15]
```

### 写内存 (STORE)

```assembly
STORE Rs2, [Rs1, #imm]
```

操作：`Mem[Rs1 + imm] ← Rs2`

示例：
```assembly
OFFSET R1, R0, #20   ; R1 = 20
STORE R3, [R1, #3]    ; Mem[23] = R3
```

## 地址对齐

Trigram-3 要求指令地址对齐：

- 指令必须从地址 0, 3, 6, 9... 开始
- 即：地址 % 3 == 0
- 禁止从地址 1, 2, 4, 5... 开始放指令

## 内存布局示例

```
逻辑地址    物理地址    内容
--------    --------    ----
-9841       0          输出端口
-9840       1          输入端口
-9839       2          保留
...
-5          9836       数据
-4          9837       数据
-3          9838       数据
-2          9839       保留

```

## 内存访问示例

### 示例 1: 基本读写

```assembly
OFFSET R1, R0, #10   ; R1 = 10
OFFSET R2, R0, #20   ; R2 = 20

STORE R1, [R2, #0]    ; Mem[20] = R1 = 10
LOAD R3, [R2, #0]     ; R3 = Mem[20] = 10
```

### 示例 2: 负地址访问

```assembly
OFFSET R1, R0, #-5   ; R1 = -5
STORE R1, [-10]       ; Mem[-10] = -5
LOAD R2, [-10]        ; R2 = Mem[-10] = -5
```

### 示例 3: I/O 操作

```assembly
; 读取输入
LOAD R1, [-9840]     ; R1 = 输入值

; 处理数据
ADD R1, R1, R1       ; R1 = R1 * 2

; 输出结果
STORE R1, [-9841]     ; 输出 R1
```

## 内存优势

对称地址空间设计的优势：

1. **寄存器值 ↔ 地址无缝映射**: 无需担心溢出或符号问题
2. **天然支持对称算法**: 如 FFT、滤波器、数学函数表
3. **简化编程模型**: 程序员可自由使用 Mem[i]，其中 i 可正可负
