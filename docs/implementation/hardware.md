# Trigram-3 硬件实现指南

## 概述

本文档提供 Trigram-3 架构的硬件实现建议和参考。

## 基本组件

### 1. 三进制逻辑单元

Trigram-3 需要实现基本的平衡三进制逻辑：

- **三进制与门** (AND): 输出 T, 0, 1
- **三进制或门** (OR): 输出 T, 0, 1
- **三进制非门** (NOT): 输入 T→1, 0→0, 1→T
- **三进制加法器**: 支持带进位的加法

### 2. 寄存器堆

- **9 个 9-trit 寄存器**
- **R0 硬连线为 0**
- **三态输出**: 用于总线连接

### 3. 算术逻辑单元 (ALU)

支持以下操作：
- 加法 (ADD)
- 减法 (通过加法和取负实现)
- 绝对值 (ABS)
- 取负 (NEG)
- 比较 (CMP)

### 4. 内存控制器

- **地址转换**: 逻辑地址 → 物理地址
- **内存访问**: 读/写操作
- **I/O 映射**: 特殊地址处理

## 数据通路

### 基本数据通路

```
       ┌─────────┐
       │  Memory │
       └────┬────┘
            │
            ▼
       ┌─────────┐
       │   ALU   │
       └────┬────┘
            │
            ▼
       ┌─────────┐
       │Registers│
       └─────────┘
```

### 指令执行流程

1. **取指**: 从内存读取 3 个字（27 trit）
2. **译码**: 解析操作码和操作数
3. **执行**: 执行指令操作
4. **写回**: 将结果写回寄存器或内存

## 时钟设计

### 时钟周期

建议使用单周期或多周期设计：

- **单周期**: 每条指令一个时钟周期
- **多周期**: 根据指令复杂度分配多个周期

### 时序要求

- **建立时间**: 寄存器输入稳定时间
- **保持时间**: 寄存器输入保持时间
- **传播延迟**: 信号传播时间

## 实现建议

### 1. 使用 FPGA

推荐使用 FPGA 实现 Trigram-3：

- **Xilinx**: Virtex, Kintex 系列
- **Altera**: Cyclone, Arria 系列

### 2. 使用 ASIC

对于大规模生产，可以考虑 ASIC 实现：

- **工艺**: 28nm 或更先进
- **面积**: 优化三进制逻辑单元
- **功耗**: 低功耗设计

### 3. 混合实现

结合 CMOS 和三进制器件：

- **CMOS**: 控制逻辑
- **三进制**: 数据通路

## 调试系统

### 概述

Trigram-3 采用直观的符号（四象、八卦）与物理按键（F1–F12 + Win）实现从上电第一拍即可干预的硬件级调试能力。

### 核心理念

- **无需定制键盘**：仅使用标准 104/108 键布局，通过固件重映射实现全部功能
- **硬件级调试**：从上电第一拍即可干预，无需操作系统支持
- **文化自明**：卦象即语义，降低学习成本

### 按键映射方案

#### 1. 太极键（●）→ `Win` 键（⊞）

**选择理由：**
- 所有 PC/Mac 键盘均有（左右各一）
- 扫描码标准（左 Win: `0xE0 0x5B`）
- 在 BIOS/裸机阶段**未被操作系统占用**
- 位置居中（空格两侧），象征"阴阳双目"，契合太极意象

**功能：**
- **开机前按住** → 进入 CPU 调试模式
- **运行中短按** → 暂停 CPU（单步准备）
- **组合使用** → 触发扩展操作（如 `Win + F3` = 强制外设输出）

#### 2. 功能键映射（F1–F12）

| 按键 | 符号 | 名称 | 功能 | 模式 |
|------|------|------|------|------|
| **F1** | ☰ | 乾 | OFFSET / 偏移 | 基础 |
| **F2** | ☷ | 坤 | HALT / 停机 | 基础 |
| **F3** | ☲ | 离 | STORE / 写内存 | 基础 |
| **F4** | ☵ | 坎 | LOAD / 读内存 | 基础 |
| **F5** | ☳ | 震 | TJUMP / 三路跳转 | 基础 |
| **F6** | ☶ | 艮 | CMP / 比较 | 基础 |
| **F7** | ☴ | 巽 | NOP / 空操作 | 基础 |
| **F8** | ☱ | 兑 | DEBUG / 进入交互模式 | 基础 |
| **F9** | ⚎ | 少阳 | ADD / 加法 | 四象 |
| **F10**| ⚍ | 少阴 | SUB / 减法（通过 ADD+NEG 实现）| 四象 |
| **F11**| ⚌ | 老阳 | ABS / 绝对值 | 四象 |
| **F12**| ⚏ | 老阴 | NEG / 取负 | 四象 |

**说明：**
- 12 个 F 键 = 8 卦 + 4 象，无冗余，全覆盖
- 基础模式对应 Trigram-3 的 8 条核心指令
- 四象模式扩展算术运算能力

### 调试模式触发机制

#### 启动时进入调试

**操作：** 开机前按住 `Win` 键（⊞）

**硬件行为：**
- 上电复位后，检测 `Win` 键状态
- 若按下 → 进入调试模式，暂停 CPU，PC = `DEBUG_ENTRY`
- 若未按 → 正常启动，跳转至 `RESET_VECTOR`

#### 运行时进入调试

**操作：** 长按 `Win` 键 2 秒

**效果：** 暂停当前执行，进入单步调试状态

### 调试模式下能做什么？

#### 1. 查看状态

自动显示：
```
[DEBUG MODE]
PC: 1T00
R0: T1T0 R1: 001T ...
```

按 `F4`（☵）+ 地址 → 查内存

#### 2. 修改状态

- `Win + F1` → 设置寄存器（如 `R2 = T10`）
- `Win + F3` → 写内存（`addr=200, val=1T`）
- `Win + F5` → 修改 PC（跳过死循环）

#### 3. 手动运算

- 按 `F9`（⚌）→ 用当前 R1/R2 执行加法，结果存 R0

#### 4. 控制执行

- `Win + 空格` → 单步执行
- `F2`（☷）→ 退出调试，重启

### 显示输出方案

**设计原则：**
- **你只提供调试数据**（PC、寄存器、内存值）
- **显示方式由系统集成商决定**

**推荐显示方式：**
- **UART** → 串口终端（推荐）
- **SPI OLED** → 本地屏幕
- **LED 阵列** → 极简指示

**你的责任：** 暴露以下信号
- `debug_pc[8:0]` - 程序计数器
- `debug_reg[7:0][8:0]` - 9 个寄存器
- `debug_mem_addr[8:0]` - 内存地址
- `debug_mem_data[8:0]` - 内存数据

> 💡 **你不管"怎么显示"，只确保"有东西可显示"。**

### 多语言支持

**默认显示：** 卦象（☰☱☲☳…）

**切换方式：**
- 连按 `Win` 键 3 次 → 切换至英文助记符（`OFFSET`, `TJUMP`, `ADD`）
- 再按 3 次 → 切回卦象

**状态栏提示：** `[☰]` 或 `[EN]`

### 调试接口信号

建议实现以下调试接口信号：

**调试寄存器：**
- **程序计数器 (PC)**: 只读，`debug_pc[8:0]`
- **指令寄存器 (IR)**: 只读，`debug_ir[26:0]`
- **状态寄存器 (SR)**: 读/写，`debug_sr[15:0]`

**调试控制：**
- **调试模式使能**: `debug_enable`
- **单步触发**: `debug_step`
- **断点地址**: `debug_breakpoint[8:0]`

**调试数据：**
- **寄存器数据**: `debug_reg[7:0][8:0]`
- **内存地址**: `debug_mem_addr[8:0]`
- **内存数据**: `debug_mem_data[8:0]`

### 调试能力总结

| 能力 | 实现方式 |
|------|--------|
| **图灵完备** | TJUMP + CMP + LOAD/STORE + ALU |
| **可调试** | Win 键入口 + 单步 + 寄存器/内存修改 |
| **可救砖** | 开机前按 Win，重写固件 |
| **跨平台** | 仅依赖标准键盘，无需小键盘 |
| **文化自明** | 卦象即语义，降低学习成本 |

> ✨ **12 个 F 键 + 1 个 Win 键 = 一台透明、可控、可传的三进制计算机。**

## 性能优化

### 1. 流水线设计

实现指令流水线以提高性能：

- **取指级**: 读取指令
- **译码级**: 解析指令
- **执行级**: 执行操作
- **写回级**: 写回结果

### 2. 缓存设计

添加指令缓存和数据缓存：

- **I-Cache**: 指令缓存
- **D-Cache**: 数据缓存
- **统一缓存**: 混合缓存

### 3. 分支预测

优化 TJUMP 指令的执行：

- **静态预测**: 预测跳转
- **动态预测**: 根据历史预测

## 测试与验证

### 1. 功能测试

验证所有指令的正确性：

- **指令测试**: 每条指令的功能
- **边界测试**: 极值情况
- **异常测试**: 错误处理

### 2. 性能测试

测试硬件性能：

- **时钟频率**: 最大工作频率
- **吞吐量**: 每秒执行指令数
- **延迟**: 指令执行时间

### 3. 可靠性测试

验证硬件可靠性：

- **温度测试**: 高低温环境
- **电压测试**: 电压波动
- **老化测试**: 长时间运行
